把项目发布流程压缩成一条命令。

XDX DevKit 是面向前端和全栈团队的发布 CLI。它把构建、环境检查、变更摘要、预览链接和回滚计划合并成可复现的 release pipeline。

特性

给开发者看的发布页，需要先展示内部结构。

这个页面不藏技术细节：安装命令、配置文件、发布状态和回滚路径都直接放在第一屏之后。

fn()

框架自动识别

识别 Next.js、Vite、Astro、Remix 和 Node 服务，自动选择构建命令和产物目录。

git

变更摘要生成

从 commit、diff 和标签生成发布说明，避免上线前临时补写 changelog。

env

环境变量扫描

对比本地、预览和生产环境，标出缺失、疑似拼写错误和未使用变量。

ci

CI 前置检查

类型检查、测试、bundle budget、路由探测和安全头在发布前统一执行。

url

预览链接归档

每一次发布生成可追踪的预览记录，便于设计、产品和 QA 在同一链接验收。

rb

回滚计划输出

上线前生成回滚命令、影响范围和最近稳定版本，避免事故时现场推理。

配置

默认能跑，也允许你精确控制。

配置文件只有发布相关的必要信息。复杂项目可以拆分多个 pipeline，普通项目可以直接使用默认值。

xdx init 在项目根目录生成 `.xdx/release.config.ts`。
xdx inspect env 对比不同环境变量并输出缺失项。
xdx release --dry-run 只生成发布计划，不执行真实部署。

发布说明

v1.0.0 发布重点

首个稳定版本聚焦三件事：可复现的发布计划、上线前检查、上线后回滚路径。

  • 发布计划文件

    每次发布生成 JSON 计划，记录 commit、命令、产物、检查结果和预览链接。

    release-plan 稳定

  • Bundle 预算门禁

    支持按路由和 chunk 设置预算。超过阈值时可阻断发布，也可以转为 warning。

    budget web

  • 自动回滚命令

    发布前输出最近稳定版本和回滚命令，事故处理不再依赖个人记忆。

    rollback ops
