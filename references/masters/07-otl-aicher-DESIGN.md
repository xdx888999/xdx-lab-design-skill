# Design System: Otl Aicher Method
# 奥托·艾舍设计方法

> 风格定位：公共系统、图标语言和色彩编码。设计要让陌生人在复杂环境中快速找到方向。
> 使用边界：参考其公共导视、运动会视觉和 pictogram 方法，不复制 Munich 1972 具体图标。

## 1. Visual Theme & Philosophy

- 核心气质：公共、清楚、模块化、人本、乐观。
- 设计目标：把复杂场景转换成可学习、可扩展的视觉系统。
- 适用场景：导视、城市服务、体育活动、机场/校园、App 图标系统、流程说明。

## 2. Color Palette & Roles

```text
Background:   #F8F7F1
Text primary: #111111
Text muted:   #5E625D
Sky blue:     #00A3E0
Grass green:  #78BE20
Signal yellow:#FFB81C
Warm orange:  #F05A28
Line:         #D7D8D2
```

## 3. Typography Rules

- Heading: `Source Sans 3`, `Inter`, `Arial`, sans-serif; weight 600.
- Body: `Source Sans 3`, `Inter`, sans-serif; weight 400.
- 信息标签要短、直接、可翻译。
- 字体系统必须支持多语言和小尺寸阅读。

## 4. Layout & Component Rules

- 所有图标必须基于统一网格、统一身体比例或统一笔画逻辑。
- 使用色彩分区来表达类别，不把颜色当装饰。
- 卡片、路标、标签、步骤条、地图图例是核心组件。
- 页面应像导视系统一样可扫描。

## 5. Imagery & Iconography

- 图标优先使用人体、方向、动作和物体的简化轮廓。
- 图形必须在 16px、24px、48px 下都可识别。
- 避免复杂阴影；用平涂和负空间表达。

## 6. Do's and Don'ts

Do:
- 先定义类别和图标网格。
- 为颜色建立明确语义。
- 测试远距离和小尺寸可读性。

Don't:
- 不要混用不同风格图标。
- 不要为了活泼改变同一系统的比例。
- 不要使用仅靠颜色区分的重要状态。

## 7. Output Behavior

- App UI：适合底部导航、空态、步骤流程和功能入口。
- 网站：适合公共服务、体育活动、学校和展馆。
- PPT：适合流程页、规则页、导览页。

## 8. Agent Prompt Guide

```text
使用 Otl Aicher 方法设计。建立统一图标网格和色彩编码，用 #00A3E0、#78BE20、#FFB81C、#F05A28 表示不同类别。界面要像公共导视系统一样清楚、可扫描、可扩展，不要复制任何 1972 奥运原图标。
```
